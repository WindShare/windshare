## 1. 目录时间语义

- 后代目录按首次扫描冻结为不可变 generation，不提供 live refresh，也不承诺点击分享时的全树快照。
- 每个目录 generation 完整提交后，其中的已选文件即可传输；不发布 provisional page。
- 冻结后源文件变化使该文件 revision 失败，不静默更新旧 generation。
- R0 为同一能力链接下的 `ShareInstance` 换代（原地刷新）预留协议语义，实现不早于 M2；
  初版分享界面明示“内容在对方首次浏览时定格，之后的修改和新增不会同步”。
- 链接就绪后默认低优先级预取所选目录根的第一层：首屏即时，第一层在分享时刻附近冻结；
  更深目录仍按首次展开冻结。纯文件分享无需预取（根事务已冻结其身份与大小）。

## 2. 小分享与连接

```text
Small = discoveryTerminal && fileCount < 30 && totalBytes < 8 MiB
```

- 只统计文件；空文件计数，目录不计数。恰好 30 个文件或 8 MiB 均为非小分享。
- 未完成或失败的目录发现不能证明为小分享，按非小分享时序处理。
- 下载点击即启动连接计时，R 合同冻结 0/8 两档：0 秒启动直连，非小分享最迟约 8 秒加入应用 relay；
  目录发现不得延后该上限。约 8 秒是对原始需求“10 秒等待”的有意收紧（接收者更早开始出字节）；
  现行实现为 0/10 两段式，随 R6 切换。TURN 升级档位待连接可用性提案批准后插入。

## 3. 发送端源文件

- 发送端不得创建内容临时文件、完整副本、staging 或本地内容快照。
- 默认优先写入排除句柄，无法取得（如源已被写模式打开）时回退可靠单调变化标识；
  两者皆不可得时直接失败。锁定随最后 lease 与 resume grace 结束释放。
- 发送端 UI 须明示：分享期间文件可能被锁定，本机修改会中断接收方下载。
- 检测到变化后永久失败该 revision，并废弃其 lease、缓存和 receiver DurableRanges。

## 4. 浏览器刷新恢复

- File System Access 后端持久化 handle 与 journal，重新授权和验证后续传。
- 不具备文件句柄时，大文件/多文件可使用接收端 OPFS staging；持久存储不可用时不得承诺恢复。
- 直接单文件/ZIP 流刷新后重新下载。刷新始终新建 ProtocolSession，不依赖 Service Worker 保活。
- OPFS 只属于接收端输出，不改变发送端禁止内容副本的规则。

## 5. 离线上传与退出

| 状态 | 服务来源 | 明确退出 |
|---|---|---|
| `LiveOnly` | 在线发送端 | 终止活动下载并使链接失效 |
| `OfflineUploading` | 在线发送端；服务器仅暂存未提交对象 | 取消上传、清理暂存、终止下载并使链接失效 |
| `OfflineCommitted` | 已提交的寄存服务，发送端可选 | 不影响现有或后续下载 |

- 上传期间同一链接始终可由在线发送端使用；未提交对象不对 receiver 可见。
- 服务器验证完整 inventory 并返回原子 commit ACK 后，UI 才能提示可以关闭软件。
- 在线下载与上传复用同一 catalog、revision 和密封对象，commit 后切换来源不得废弃已验证范围。
- 上传中源文件漂移会使 commit 失败。寄存服务不取得解密密钥或 sender 私钥。
- 只有意外断线可进入具名恢复宽限；明确停止或退出不得恢复在线路由。

## 6. 元数据隐私

文件名、精确大小和文件标识不得明文暴露。允许 relay 从密文长度、时序和传输量推断近似信息；
本阶段不要求 padding 或流量隐藏。

## 7. 预览粒度

- 首版预览仅由用户对单个文件的显式点击触发，不提供缩略图墙；批量预览列为 M2/M3 评估项。
- R0 的 lease/句柄/配额合同须为批量预览预留余量，避免未来回改已冻结预算。
