# R8 Performance Baseline

This baseline is for regression triage, not a portable speed promise. It records only the final implementation snapshot and explicitly excludes blocked or superseded measurements.

## R8-TREND-v1

- Each aggregate requires five independent raw samples. Nearest-rank p50 is sorted sample 3 and p95 is sorted sample 5.
- Semantic and resource invariants are hard gates: exact bytes/hashes, zero forbidden filesystem work, bounded descriptors/allocations/DOM/buffers, exact-once fetch, cleanup, and capability-accurate results. Time and throughput are descriptive.
- Compare only the same workload, runner, toolchain, capability set, and host class. A greater-than-25% p50 or p95 degradation becomes a warning only after two consecutive comparable runs; it never overrides an immediate hard-invariant failure.

### Comparison boundary

The July 17 source digest `ee5f1d0a18b023ef2e2679c5777524d67e2fe3a4cc61c8abe56f9da5cd90b1c3` and its 17-aggregate/85-sample record are historical, not a comparator or current acceptance evidence. The current contract adds relay registration, uses a different source digest, and lacked complete physical-RAM metadata. Apparent single-run latency increases above 25% therefore do **not** satisfy the two-consecutive-comparable-run warning rule. The Pion 1,114,112-byte peak-buffer ceiling was outside this Go-only run and remains unevaluated by current performance evidence.
